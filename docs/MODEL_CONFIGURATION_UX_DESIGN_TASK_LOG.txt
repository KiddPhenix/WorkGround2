服务暂不可用时，显示\"重试\"按钮，同时展示上次成功拉取的模型列表（带\"数据可能已过期\"提示）。\n+- 后台刷新模型列表失败时，沿用上次可用列表，并在服务卡片上显示黄色提示。\n+\n+### 恢复规则\n+\n+- 删除连接前若该服务为默认模型来源，弹出替代选择面板，用户必须选择新默认模型后才能删除。\n+- 切换默认模型到未连接的服务时，不允许操作并提示。\n+\n+---\n+\n+## 11. 数据与 UI 边界\n+\n+遵循全局设计规范第 11 章的归属原则：\n+\n+- **配置状态进入 ConfigStore**（待新增）：连接状态、模型列表、默认模型、高级配置等由 ConfigStore 统一管理。\n+- **UI 订阅状态**：页面组件通过 `useConfigStore` 读取状态，不直接持有可变配置副本。\n+- **回包不直接操作 Panel**：API Key 校验、模型拉取等网络回包先更新 ConfigStore，UI 通过订阅反应变化。\n+- **状态入口收敛**：连接状态的修改统一通过 ConfigStore 的方法（`addProvider`、`removeProvider`、`setDefaultModel`
